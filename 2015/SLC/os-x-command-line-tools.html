OS X Command Line Tools

OS X is built on Darwin, an operating system that has its origins in Mach and FreeBSD. It is a full-fledged UNIX system.

Yes, this is old school.

Typing is faster than a GUI.

The Internet is not always available when you want to get work done.

Finding commands

  • man: Display manual page
    • man <command>
    • man -k <keyword>
      Search for a keyword in short descriptions of system commands.

Figuring out command syntax


jmdodd@milano.local:~$ man whois
				

WHOIS(1)                  BSD General Commands Manual                 WHOIS(1)

NAME
     whois -- Internet domain name and network number directory service

SYNOPSIS
     whois [-aAbdgiIlmQrR6] [-c country-code | -h host] [-p port] name ...

DESCRIPTION
     The whois utility looks up records in the databases maintained by several
     Network Information Centers (NICs).

     The options are as follows:

     -6      Use the IPv6 Resource Center (6bone) database.  It contains net-
             work names and addresses for the IPv6 network.

     -A      Use the Asia/Pacific Network Information Center (APNIC) database.
             It contains network numbers used in East Asia, Australia, New
             Zealand, and the Pacific islands.

     -a      Use the American Registry for Internet Numbers (ARIN) database.

<command> [optional flags]

Disk Utilities

  • du: Display disk usage statistics
    • du -d 1
      Depth of one directory in current directory.
    • du -s *
      Side of files and directories in current directory
  • df: Display free disk space
    • df

Eventually, you're going to do something interesting involving a log file or a download process or something creative that will eat your disk space.

File Utilities

  • pwd: Print working directory
  • cd: Change directory
    • cd ~
      Change to your home directory.
    • cd ~/Downloads
      Change to your Downloads directory.
  • ls: List your files
    • ls -la
      List all the details of all the files in the pwd, including dotfiles.
    • ls -1
      List all the files in the pwd in one column.
  • cat: Concatenate a file to your screen
    • cat <file>
  • head: Output the first part of a file
    • head <file>
  • tail: Output the last part of a file
    • tail -<number> <file>
      Output the last <number> lines of a file.
    • tail -f <file>
      Follow the file as it grows.
  • less: Scroll backward and forward through a file
    • less <file>
  • wc: Word count
    • wc -l <file>
      Count newlines.
    • wc -L <file>
      Print length of longest line.
    • wc -w <file>
      Print the word count.
    • wc -m <file>
      Print the character count.
  • grep: Look for a string in file(s)
    • grep -i <pattern> <file(s) or pattern>
      Case-insensitive search.
    • grep -v <pattern> <file(s) or pattern>
      Find lines that do not match <pattern>.
    • grep -R <pattern> <file(s) or pattern>
      Recursively search subdirectories.
  • find: Find types of files
    • find -type d
      Find directories.
    • find -type f
      Find regular files.
    • find -type l
      Find symbolic links.
  • locate: Find a file by name
    • locate <pattern>
      Find pathnames that match the given pattern.
  • which: Find the source path of a command
    • which <command>

Text Utilities

  • diff: Find the differences between two files
    • diff <file1> <file2>
  • vimdiff: Display the differences between two files using vim to highlight findings
    • vimdiff <file1> <file2>
  • sort: Sort lines
    • sort <file>
      Sort the lines in a file.
    • sort -f <file>
      Ignore case.
    • sort -n <file>
      Sort numerically.
  • uniq: Find unique lines in a file
    • uniq <file>
  • xmllint: Check a file for valid XML
    • xmllint <file>

Connectivity and Network

  • ping: Ping a network host
    • ping wordpress.com
  • traceroute: Print the route to a network host
    • traceroute ns1.wordpress.com
  • whois: Internet domain name directory service
    • whois wordpress.org
    • whois wordpress.com
    • whois "domain wordpress.com"
    • whois -h whois.markmonitor.com wordpress.com
  • dig: DNS record lookup utility
    • dig @ns1.wordpress.com wordpress.com ANY
    • dig @ns1.wordpress.com wordpress.com MX
  • host: IP lookup utility
    • host 198.181.116.9
    • host wordpress.com
  • curl: Transfer a URL
    • curl -v https://wordpress.com/
    • curl -I https://wordpress.com/
    • curl -O https://wordpress.org/latest.zip

Processes

  • ps: List processes
    • ps -auxwww
      List complete information about all processes, including those running in the background
  • kill: Kill a given process
    • kill -9 <pid>
      Non-catchable, non-ignorable kill signal.
    • kill -HUP <pid>
      Hang-up signal.
  • &: Background a process
    • curl -s -O URL_OF_HUGE_FILE &
      Background the download of a large file.

Pipes and Redirection

  • ;: Run multiple commands in sequence, separating them with ;
    • mkdir tmp; touch tmp/lockfile;
      Create a directory and a lockfile in that dir.
  • |: Pipe the output from one command to another command
    • ls -1 | grep -v jpg
      Find all the files not containing 'jpg' in the pwd.
    • tail -f php_error.log | grep 10.0.1.1
      Display lines that match 10.0.1.1.
  • >: Overwrite file contents
    • ls -1 > files.txt
      Save a list of pwd files in files.txt.
  • >>: Append file contents
    • ls -1 >> files.txt
      Append the list of pwd files to files.txt.

curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar

chmod +x wp-cli.phar

Using MAMP? In ~/.bash_profile:


# Use MAMP version of PHP
PHP_VERSION=`ls /Applications/MAMP/bin/php/ | sort -n | tail -1`
export PATH=/Applications/MAMP/bin/php/${PHP_VERSION}/bin:$PATH

# Export MAMP MySQL executables as functions
# Makes them usable from within shell scripts (unlike an alias)
mysql() {
    /Applications/MAMP/Library/bin/mysql "$@"
}
mysqladmin() {
    /Applications/MAMP/Library/bin/mysqladmin "$@"
}
export -f mysql
export -f mysqladmin

Then run source ~/.bash_profile

Source: How to override the path of PHP to use the MAMP path?
